# Proxy Guide

## Где настраивать

1. Открой `Proxy Manager` в главном меню.
2. Включи прокси.
3. Выбери режим работы.
4. Заполни файл `proxies.txt`.

## Поддерживаемые форматы

```text
socks5://127.0.0.1:9050
socks5://user:password@127.0.0.1:9050
http://127.0.0.1:8080
127.0.0.1:9050
127.0.0.1:9050:user:password
socks5:127.0.0.1:9050:user:password
```

Если тип не указан, по умолчанию используется `socks5`.

## Режимы

- `chunked_round_robin` — рекомендуемый режим. Аккаунты делятся на прокси равномерными пачками, но обрабатываются по кругу.
- `round_robin` — каждый следующий аккаунт получает следующий прокси по кругу.
- `sticky_hash` — каждая сессия стабильно закрепляется за одним и тем же прокси.
- `single` — все аккаунты работают через первый прокси из списка.

## Как работает основной режим

Пример:

- 100 аккаунтов
- 10 прокси

В режиме `chunked_round_robin` каждый прокси получает примерно по 10 аккаунтов, но порядок обработки будет не `10 подряд на proxy1`, потом `10 подряд на proxy2`, а по кругу:

```text
acc01 -> proxy1
acc11 -> proxy2
acc21 -> proxy3
...
acc02 -> proxy1
acc12 -> proxy2
acc22 -> proxy3
```

Так нагрузка распределяется мягче и один прокси не получает длинную подряд идущую серию аккаунтов.

## Как работают остальные режимы

### `round_robin`

Каждый следующий аккаунт получает следующий прокси по кругу.

Пример:

```text
acc01 -> proxy1
acc02 -> proxy2
acc03 -> proxy3
acc04 -> proxy1
acc05 -> proxy2
acc06 -> proxy3
```

Это самый простой режим равномерной ротации. Он хорошо размазывает нагрузку, но не держит за аккаунтом постоянный прокси.

### `sticky_hash`

Для каждой сессии считается стабильная привязка к прокси по имени файла сессии. Если список прокси не меняется, одна и та же сессия будет получать тот же прокси между запусками.

Пример поведения:

```text
111111.session -> proxy2
222222.session -> proxy5
333333.session -> proxy2
111111.session -> proxy2  (и в следующем запуске тоже)
```

Это удобно, если хочется, чтобы аккаунт "жил" на одном и том же прокси, а не прыгал между ними. Важно: если изменить количество прокси или их порядок в `proxies.txt`, привязки могут поменяться.

### `single`

Все аккаунты идут через первый прокси из списка.

Пример:

```text
acc01 -> proxy1
acc02 -> proxy1
acc03 -> proxy1
acc04 -> proxy1
```

Этот режим полезен для быстрой проверки одного прокси или если нужен полностью единый маршрут для всех аккаунтов.

### Без прокси

Если прокси отключены в `Proxy Manager`, все операции идут напрямую, без использования `proxies.txt`.

Пример:

```text
acc01 -> direct
acc02 -> direct
acc03 -> direct
```

Это обычный режим работы проекта, когда прокси вообще не участвуют.

## Где смотреть план распределения

После массовых операций последний план сохраняется в:

```text
results/proxy_plans/*.json
```

Там видно:

- какой аккаунт на какой прокси попал;
- порядок обработки;
- сколько аккаунтов получил каждый прокси.
